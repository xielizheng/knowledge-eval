# Knowledge Eval

知识库效果自动化评测原型，用于评测批次管理、自动评分分析和人工校验回流方案验证。

## 页面版本

| 文件 | 版本 | 定位 |
| --- | --- | --- |
| `知识库效果自动化评测V0.5.html` | V0.5 | 一期降本方案：批次管理、CSV 下载、Excel 人工校验、CSV 导入回流，不提供单条在线人工评测 |
| `知识库效果评测V1.0.html` | V1.0 | 完整交互方案：聚合指标、趋势、自动归因、评测流水和明细人工修正 |
| `index.html` | 当前默认页 | 当前默认展示版本，重要版本仍以独立 HTML 文件保留 |

页面均为单文件静态原型，可直接在浏览器中打开。

## 版本管理约定

- 重要产品节点使用语义化版本号：`v主版本.次版本.修订号`，例如 `v1.1.0`。
- 每次重点改动先更新独立版本文件和 `CHANGELOG.md`，再提交 Git commit。
- 稳定节点创建 Git tag，便于对比和回滚。
- 小范围文案或样式修复递增修订号；新增功能递增次版本号；流程或数据结构发生不兼容变化时递增主版本号。

常用命令：

```bash
# 查看所有稳定版本
git tag --list

# 查看某个版本
git show v1.0.0

# 基于历史版本创建回滚分支
git switch -c rollback/v1.0.0 v1.0.0
```
